mechanism_status_notes: The live validator scores miners on GPU hardware inventory and container uptime
  (a proof-of-resources mechanism) in evaluate_miners.get_rewards, normalizes to sum=1, and EMAs into
  on-chain weights. The README markets an aspirational "AI Agent Launchpad / TAOPump" economy with agent
  tokens, creators and users, but none of that scoring is present in code — only hardware/uptime scoring
  is implemented. No burn UID is present.
subnet:
  name: SN108 Internet of Intelligence (Agent Launchpad)
  owner_repo: subnet108__internet-of-intelligence
  netuid: 108
  implementation_lang: python
task:
  summary: A miner runs a server (queried at the config endpoint) that reports its GPU inventory and the
    uptime/status of its running containers ("pods"). The validator queries a random sample of miners,
    verifies the reported identity (ip/port/coldkey/hotkey/nonce/signature/timestamp), then scores each
    miner on (a) its share of total GPUs, (b) its share of total container uptime, and (c) a bonus for
    long-running containers, all multiplied by a GPU-model rarity weight. There is no AI/agent task being
    evaluated in code; reward is hardware capacity and liveness.
  submission_format:
  - compute_service
  - api_response
scoring_signals:
- name: gpu_count_share
  metric_kind: hardware_spec
  direction: higher_is_better
  weight: 0.5
  normalization: min_max
  notes: score_a = len(r["gpu"]) / total_gpus across all valid responses; weighted by GPU_CORES_SCORE
    = 0.5.
- name: container_uptime_share
  metric_kind: uptime
  direction: higher_is_better
  weight: 0.3
  normalization: min_max
  notes: score_b = sum of uptimes of containers with status==1, divided by total uptime across all valid
    responses; weighted by POD_RUN_TIME_SCORE = 0.3.
- name: long_running_bonus
  metric_kind: uptime
  direction: higher_is_better
  weight: 0.1
  normalization: none
  notes: score_c = 1/total_long_run if a miner's average container uptime exceeds POD_RUN_TIME_AVG_DAY
    (604800s = 7 days); weighted by POD_RUN_TIME_AVG_SCORE = 0.1.
- name: gpu_model_rarity_multiplier
  metric_kind: hardware_spec
  direction: higher_is_better
  weight: null
  normalization: none
  notes: gpu_weight = sum of GPU_MODEL_RATES per GPU model; applied as a multiplicative (1 + gpu_weight)
    factor on the additive score, not an additive term. Rates literal in const.py (e.g. H100 80GB HBM3
    = .10).

anti_gaming:
- kind: liveness_check
  enforcement: rejection
  description: Responses are dropped (scored 0) unless status is true and the reported ip/port/coldkey/hotkey
    match the metagraph axon, nonce matches, signature verifies against public.key, and timestamp is within
    10000 ms of now.
- kind: deduplication
  enforcement: rejection
  description: If more than one active axon shares the same IP (active_ip_count[ip] > 1), that miner's
    response is invalidated (set to None / score 0).
- kind: hardware_validation
  enforcement: barrier
  description: GPU_MODEL_RATES whitelists rewarded GPU models; unknown models contribute 0 to gpu_weight.
    Only containers with status==1 count toward uptime.
