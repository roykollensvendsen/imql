mechanism_status: partial_burn
mechanism_status_notes: Live scoring is active but heavily burn-weighted. A configurable burn_fraction
  (default 0.65) is sent to hardcoded burn UID 59 whenever miners qualify; if no miner clears the quality
  threshold, 100% of emissions burn to UID 59. With UAV_grading enabled (default True), kept emissions
  are split 10% KAV / 90% UAV reputation, so most reward flows through an off-chain reputation snapshot
  rather than the documented online name-variation quality scoring.
subnet:
  name: MIID Subnet 54 - Identity Testing Network
  owner_repo: yanez-compliance__MIID-subnet
  netuid: 54
  implementation_lang: python
task:
  summary: Miners receive seed identities (name, date of birth, address) plus a threat-scenario spec and
    must generate name variations (and DOB/address variations) that simulate identity-evasion techniques
    used to bypass KYC/sanctions screening. Validators grade variations on phonetic and orthographic similarity,
    count/uniqueness/length, rule-compliance, and address/DOB validity.
  submission_format:
  - signals

  metric_kind: similarity
  direction: target_band
  weight: 0.6
  normalization: none
  notes: Combined phonetic (Soundex/Metaphone/NYSIIS) and orthographic (Levenshtein) similarity scored
    against target distribution bands (Light/Medium/Far). similarity_weight=0.60 in MIID_REWARD_WEIGHTS,
    applied per-part in calculate_part_score.
- name: count_match
  metric_kind: composite
  direction: target_band
  weight: 0.15
  normalization: none
  notes: Score for producing the expected number of variations within an adaptive tolerance; exponential-decay
    penalty for deviation. count_weight=0.15.
- name: uniqueness
  metric_kind: composite
  direction: higher_is_better
  weight: 0.1
  normalization: none
  notes: uniqueness_weight=0.10; ratio of sufficiently-distinct variations after clustering.
- name: length_reasonableness
  metric_kind: composite
  direction: target_band
  weight: 0.15
  normalization: none
  notes: length_weight=0.15; adaptive length-ratio score vs original name.
- name: rule_compliance
  metric_kind: composite
  direction: higher_is_better
  weight: 0.2
  normalization: none
  notes: rule_compliance_weight=0.20 in MIID_REWARD_WEIGHTS; final per-name score is base_weight*base_score
    + rule_compliance_weight*rule_compliance_score with base_weight = 1.0 - rule_compliance_weight.
- name: address_validity
  metric_kind: composite
  direction: higher_is_better
  weight: 0.7
  normalization: none
  notes: In final per-miner reward, components are weighted quality_weight=0.2, dob_weight=0.1, address_weight=0.7.
    Address scored via Nominatim bounding-box area buckets (1.0/0.9/0.8/0.7/0.3).
- name: dob_validity
  metric_kind: composite
  direction: higher_is_better
  weight: 0.1
  normalization: none
  notes: dob_weight=0.1 in final reward composition (_grade_dob_variations).
- name: name_variation_quality
  metric_kind: composite
  direction: higher_is_better
  weight: 0.2
  normalization: none
  notes: quality_weight=0.2; the average per-name variation quality score (the similarity/count/ uniqueness/length/rule
    composite from calculate_variation_quality) folded into the final per-miner reward alongside DOB and
    address.

  trust_model: unknown

  method: rank_based
  composition: hybrid
  normalization: l1
  temperature: null
  decay_rate: 0.05
  min_weight_floor: null
  burn_allocation:
    enabled: true
    address_or_uid: '59'
    fraction: 0.65
    dynamic: true
    notes: burn_fraction default 0.65 to hardcoded burn_uid 59 when miners qualify; kept rewards rescaled
      to keep_fraction (1 - burn_fraction). 100% burn (fraction 1.0) when no miner clears the quality

  notes: 'Per-name quality is a weighted_sum of similarity/count/uniqueness/length plus a separate rule-compliance

    miners, quality_threshold (0.6) gates them, qualified miners get exp(-decay_rate*rank) blended with

    reweighting.'

    kind: ema

anti_gaming:
- kind: deterministic_check
  enforcement: penalty
  description: Phonetic algorithm subset and weights are randomized (deterministically seeded per name)
    to reduce overfitting to a single encoding; variations with >2 consecutive identical letters are filtered
    with up-to-20% penalty; very-low-similarity scores (<0.2) reduced to 10%.
- kind: deduplication
  enforcement: penalty
  description: Address-duplication penalty (normalized, capped at 0.2) and DOB/address normalization-based
    duplicate counting in cheat_detection / get_name_variation_rewards.
- kind: plagiarism_detection
  enforcement: penalty
  description: detect_cheating_patterns computes pairwise overlap/Jaccard similarity between miners; large
    collusion groups (members with score < 0.95) get a direct penalty (penalty_value = 0.75); high cross-miner
    overlap penalized via overlap/Jaccard.
- kind: rate_limit
  enforcement: barrier
  description: Quality threshold (0.6) plus top_miner_cap (50) gate which miners receive any ranking reward;
    non-qualifying miners get 0 and emissions burn.
sub_competitions:
  structure: multi_mechanism
  stages:
  - name: KAV (Known Address Variation) online quality scoring
    role: main
    advancement_rule: quality_score >= quality_threshold (0.6) and global_rank < top_miner_cap (50)
    multipliers:
      kav_weight: 0.1
  - name: UAV (Unknown Attack Vectors) reputation-weighted rewards
    role: other
    advancement_rule: rep_score > 0 in off-chain rep_data snapshot
    multipliers:
      uav_weight: 0.9
  count: 2
  notes: UAV_grading config flag (default True) enables the two-track KAV+UAV pipeline; when disabled,
    KAV-only scoring with burn applied directly.
