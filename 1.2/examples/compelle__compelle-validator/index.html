mechanism_status: active
mechanism_status_notes: Live scoring drives weights. Each epoch the validator runs a Swiss-pairing LLM
  debate tournament across eligible miners, updates Elo, derives softmax weights from Elo and calls set_weights.
  A tiny EPS_BUDGET=0.001 fraction is reserved for placeholder ("epsilon") miners by registration-age;
  the remaining 0.999 is scaled across real miners. No burn UID. Epsilon-only epochs (no real miners)
  skip set_weights entirely to preserve the rate-limit window rather than burning.
subnet:
  name: Compelle
  owner_repo: compelle__compelle-validator
  netuid: 82
  implementation_lang: python
task:
  summary: Compelle is an adversarial-persuasion subnet. Miners commit a written debate strategy on chain
    (inline text or a revisioned GitHub gist reference). Each epoch the validator pairs miners in Pro/Con
    LLM debates over a single shared topic (motion), runs the debaters and a judge panel via Chutes-hosted
    LLMs, scores outcomes with Elo, and converts Elo to weights. The miner whose strategy wins more debates
    earns higher Elo and thus higher weight.
  submission_format:
  - commitment_hash
  - content_link
  extensions:
    commitment_detail: The on-chain commitment is either inline strategy text (<=65536 bytes) or a 'gist:<id>/<revision>'
      reference to an immutable GitHub gist revision. Modeled as commitment_hash (on-chain commitment)
      + content_link (gist URL fetched off-chain).
scoring_signals:
- name: debate_win_elo
  metric_kind: win_rate
  direction: higher_is_better
  weight: 1.0
  normalization: softmax
  notes: Sole scoring signal. Pro/Con debate outcomes (win/loss/draw) update an Elo rating per hotkey
    via standard Elo (k_factor=32, initial=1000, expected = 1/(1+10**((rb-ra)/400))). Per-epoch weights
    derived from Elo via softmax with temperature=100. Win counts also drive Swiss pairing.
ground_truth_sources:
- kind: llm_judgment
  description: Debate winners are decided either by concession (a debater starts a message with the Greek
    delta symbol and >=50 chars) or by a judge. The judge is a majority vote across a configured panel
    of LLM models (judge_panel); when the panel can't reach a clear majority it falls back to a single
    judge with primary+fallback model chain.
  source_identifier: 'Chutes LLM API (judge_panel: Qwen3-235B, DeepSeek-TNG-R1T2, MiniMax-M2.5)'
  trust_model: oracle
  notes: 'Non-deterministic: debater and judge calls use non-zero temperature (debate temp=0.6, judge
    temp=0.3 + 0.2*attempt). Validators converge over many epochs via Yuma consensus rather than per-epoch
    agreement.'

  burn_allocation:

    address_or_uid: null
    fraction: null
    dynamic: false
    notes: No burn UID. EPS_BUDGET=0.001 of weight is reserved for placeholder ("epsilon") miners distributed

  notes: 'Elo.weights(temperature=100) softmaxes ratings: e[k]=exp((v-max_r)/temp), weight=e[k]/sum(e).

weight_setting:
  cadence: rate_limited
  method: 'Once per epoch loop iteration: build Elo softmax weights for real miners, add age-weighted

    + rate_limit) and skips if within the window. Re-queries WeightsVersionKey per attempt; up to 3 attempts
    with 5s backoff. bittensor 10.x handles commit-reveal automatically (wait_for_revealed_execution=False).'
  on_chain_call: set_weights
  tempo_or_interval: tempo_blocks=360 (tournament/topic bucket); epoch loop sleeps epoch_seconds=4320s
  smoothing:
    kind: other
    kind_other: persistent_elo
    alpha: null
    window: null
    notes: 'No EMA/SMA on the final weight vector. Smoothing is implicit via persistent cumulative Elo
      carried across epochs (elo_state.json): ratings accumulate across tournaments rather than being
      recomputed fresh each epoch, so weights move gradually. Tournament idempotency: one tournament per
      tempo (last_completed_tempo.txt); within the same tempo the cached Elo is reused.'
anti_gaming:
- kind: rate_limit
  enforcement: barrier
  description: Chain WeightsRateLimit pre-checked before set_weights; calls within the window are skipped.
    Tournament is run at most once per tempo (current_tempo <= last_tempo => skip).
- kind: commit_reveal
  enforcement: rejection
  description: 'Strategies referenced via gist must pin an immutable revision: regex ^gist:([0-9a-f]{20,40})/([0-9a-f]{40})$.
    Mutable ''gist:<id>'' refs are rejected so a miner cannot swap their committed strategy after the
    fact. Multi-file gists rejected (non-deterministic file selection).'
- kind: deterministic_check
  enforcement: rejection
  description: 'Eligibility window: a commitment is only eligible if 0 <= commitment_block - registration_block

    from real scoring. Stale Elo for deregistered or structurally-malformed-gist hotkeys is pruned each
    tournament.'
- kind: code_inspection
  enforcement: penalty
  description: Judge and debater system prompts include prompt-injection hardening (treat strategy/transcript

    statistics, citations, quotes, and events.
